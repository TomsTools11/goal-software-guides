# Getting Started with Claude Cowork

Your complete guide to multi-step task automation with Claude Desktop — designed for the GOAL Platform team.

---

## What is Claude Cowork?

Cowork uses the same agentic architecture that powers Claude Code, now accessible within **Claude Desktop** — no terminal required. Instead of responding to prompts one at a time, Claude can take on complex, multi-step tasks and execute them on your behalf.

With Cowork, you can describe an outcome, step away, and come back to **finished work** — formatted documents, organized files, synthesized research, interactive dashboards, and more.

<TipCallout variant="info" title="Availability">
Cowork is available as a research preview for **paid plans only**: Pro, Max, Team, and Enterprise. It requires the Claude Desktop app on **macOS or Windows** (x64 only). Cowork is **not** available on web or mobile.
</TipCallout>

<AnimatedDemo type="cowork-setup" />

The key shift from Chat to Cowork is this: **Chat mode** is for quick Q&A, brainstorming, and one-shot tasks. **Cowork mode** is for multi-step projects where Claude needs to read files, create deliverables, and work autonomously over extended periods.

## Key Capabilities

Cowork isn't just a chatbot with file access — it's a fundamentally different way of working. Here are the five capabilities that set it apart.

<InfoCard title="Direct Local File Access" icon="📂">
Claude can **read from and write to your local files** without manual uploads or downloads. Point it at a folder and it can analyze spreadsheets, read documents, examine code, and create new files — all directly on your file system. No more copy-pasting content into chat windows.
</InfoCard>

<InfoCard title="Sub-Agent Coordination" icon="🔀">
For complex work, Claude **breaks tasks into smaller subtasks** and coordinates parallel workstreams. One sub-agent might analyze data while another researches a topic and a third writes a summary. This parallelism means complex projects complete much faster than sequential chat conversations.
</InfoCard>

<InfoCard title="Professional Outputs" icon="📊">
Claude generates **polished, production-ready deliverables**: Excel spreadsheets with working formulas, PowerPoint presentations, formatted documents, HTML dashboards, and more. These aren't rough drafts — they're files your team can use immediately or hand off to clients.
</InfoCard>

<InfoCard title="Long-Running Tasks" icon="⏱️">
Cowork can work on complex tasks for **extended periods** without the conversation timeouts or context limits that affect chat mode. Start a research task, walk away, and come back to a complete analysis. Claude maintains focus and context throughout.
</InfoCard>

<InfoCard title="Scheduled Tasks" icon="📅">
Create tasks that run **on-demand or automatically on a recurring schedule**. Type `/schedule` in any Cowork session to set up recurring tasks. Check the "Scheduled" section in the left sidebar to view, create, and manage your scheduled work.
</InfoCard>

### What to Expect During Execution

- **Progress indicators** show what Claude is doing at each step
- **Transparency** — Claude surfaces its reasoning and approach so you can follow along

## Real-World Use Cases

Here's how teams are using Cowork to transform their workflows.

<Accordion title="Analyze Entire Folders of Documents" defaultOpen>
Point Claude at a folder of documents — call transcripts, survey responses, support tickets, quarterly reports, proposals — and ask it to categorize, summarize, and extract insights.

**Example:** Point Claude at 321 podcast transcripts → categorize guests by topic, rank AI sentiment, pull key quotes → get an interactive, filterable dashboard as output.

**Also works for:** customer call transcripts, competitor research, proposal analysis, and support ticket categorization.
</Accordion>

<Accordion title="Research, Compare, and Make Decisions">
Ask Claude to research options, compare alternatives, and organize findings into a structured format.

**Example:** "Find podcast studios in Austin, compare pricing, amenities, and availability, then create a comparison spreadsheet and begin filling out the top three booking forms."

**Also works for:** supplier evaluations, software comparisons, venue scouting, market research, and vendor selection.
</Accordion>

<Accordion title="Build Interactive Dashboards and Reports">
Claude can create filterable dashboards with categories, sentiment scores, charts, and data tables — output as interactive HTML files that open in any browser.

**Example:** Analyze a CSV of customer feedback → build a dashboard with sentiment scores, topic categories, and filterable views → deploy via Vercel for team access.

**Also works for:** executive reports, client deliverables, data visualizations, and KPI dashboards.
</Accordion>

<Accordion title="Run Multiple Projects Simultaneously">
Open as many Cowork threads as you need. Each thread operates independently — one analyzes data, another researches a market, a third builds a presentation.

**Key technique:** Stack follow-up instructions while Claude is still working. Don't wait for one task to finish before queuing the next.

*"I can walk away from my computer for hours while all of these threads are managing all of these stacked up tasks."* — Allie K. Miller, Forbes
</Accordion>

<Accordion title="Hand Off Outputs to Your Team">
Everything Claude creates is a real file on your computer. Download apps, move files into VS Code, hand to an engineer, or connect to Vercel for deployment.

**The workflow:** Claude builds the first version → you review and refine → your team finishes the last mile. This dramatically accelerates the 0-to-1 phase of any project.
</Accordion>

## Permissions, Limits, and Best Practices

<TipCallout variant="warning" title="Security: Know the Boundaries">
Cowork runs **directly on your computer** and can make real changes to your files. Code executes safely in an isolated space, but outputs are written to your local file system. Conversation history is stored locally and is **not** captured in Audit Logs, Compliance API, or Data Exports. **Do not use Cowork for regulated workloads.**
</TipCallout>

### Permissions and Security

- Cowork runs locally on your machine — not in the cloud
- Claude can read and write files in the folders you grant access to
- Deletion requires explicit permission — Claude won't silently remove files
- Conversation history is stored locally, not subject to Anthropic's data retention
- Activity is **not** captured in enterprise Audit Logs or Compliance APIs

### Usage Limits

Cowork consumes more usage allocation than standard chat — it's compute-intensive and uses more tokens. To manage limits effectively:

- **Batch related work** into single sessions rather than many small tasks
- Use standard **Chat mode for simple Q&A** that doesn't require file access
- Monitor your usage in **Settings → Usage**

### Current Limitations

- Desktop app must remain open during tasks
- Scheduled tasks only run while your computer is awake
- Excel/PowerPoint integration is Mac-only (research preview)
- Windows arm64 not supported
- Not available on web or mobile

### Global and Folder Instructions

**Global Instructions** are standing instructions that apply to every Cowork session. Navigate to Settings → Cowork → Edit Global Instructions. Use these to specify your preferred tone, output format, and background on your role.

**Folder Instructions** provide project-specific context when you select a local folder. Claude can also update these on its own during a session as it learns about your project.
